request-my-ehi

A Claude Code skill that helps patients request their complete Electronic Health Information export from any healthcare provider.

Download skill.zip View on GitHub

What is EHI Export?

Every certified EHR system in the US has been required to support single-patient EHI Export since December 31, 2023. This feature produces a bulk export of all structured data in your patient record -- far more complete than a CCDA, patient portal download, or standard records release.

Most patients don't know it exists. This skill changes that.

What It Does

Identifies Your EHR

Detects your provider's EHR system from portal URLs, web search, or a database of 70+ certified vendors.

Finds the Right Form

Locates your provider's authorization form online, or uses a generic HIPAA-compliant fillable PDF.

Fills Everything Out

Programmatically fills the form using pdf-lib's form field API. Embeds your signature if provided.

Vendor-Specific Appendix

Generates a one-page appendix citing the exact product, export format, documentation URL, and legal basis.

How It Works

  1. Understand your situation and explain EHI Export
  2. Identify your provider's EHR system
  3. Collect your details (or extract from an uploaded file)
  4. Find the provider's authorization form
  5. Find fax/mailing address for submission
  6. Fill out the form programmatically
  7. Embed your signature (optional)
  8. Generate a vendor-specific appendix
  9. Merge into a ready-to-submit 2-page PDF
  10. Help you actually submit the request

Installation

Option A: Git clone

git clone https://github.com/jmandel/request-my-ehi.git ~/.claude/skills/request-my-ehi

Option B: Download

Download skill.zip and extract into ~/.claude/skills/:

cd ~/.claude/skills && unzip ~/Downloads/skill.zip

Prerequisites

  • Claude Code CLI
  • Node.js >= 18
  • pdf-lib (installed automatically by the skill)

Usage

Just tell Claude Code what you need:

I need to request my complete medical records from Associated Physicians of Madison.
Help me get my EHI export. My provider uses athenahealth.
I want all my health data. My patient portal is at mychart.myhealthsystem.org.

Supported Vendors

The skill includes a database of 70+ certified EHR vendors with detailed export information -- product names, export formats, documentation URLs, entity/field counts, and quality grades. Vendors include Epic, athenahealth, Cerner/Oracle Health, NextGen, eClinicalWorks, Veradigm, Elation, DrChrono, and many more.

Data sourced from the EHI Export Analysis project.

HIPAA Right of Access (45 CFR 164.524) -- patients can request their PHI in electronic form.

21st Century Cures Act (45 CFR Part 171) -- declining to use a certified export feature may constitute information blocking.

ONC Certification 170.315(b)(10) -- EHI Export required since December 31, 2023.
